Detailed Description

Content of a remote notification.

The maximum size of the payload is 256 bytes.

The payload can include app-specific key/value pairs that trigger a special behavior in your app, such as displaying a message to the user on launch. These key/value pairs are stored in the extraKeys and extraValues properties.

Note: Some properties are used only on Android or iOS, but not both. The Mobage server automatically discards any parameters that are not supported by the target user's device. Discarded parameters do not count towards the 256-byte limit.

Since:

- (NSString*) collapseKey
readwriteatomicstrong

An identifier that causes newer notifications with the same identifier to be discarded.

For example, if a user receives four notifications with the same identifier, only the newest notification will be displayed on the user's device. Used only on Android.

Since:
2.0
- (NSArray*) extraKeys
readwriteatomicstrong

The app-specific keys to include in the payload.

Each key must be represented as an NSString. The following key names are reserved and cannot be used:

  • badge
  • collapseKey
  • extras
  • iconUrl
  • message
  • sound
  • style

2.0
