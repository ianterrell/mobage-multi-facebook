Detailed Description

Interface for purchasing virtual in-app items.

To complete a purchase, your app typically follows these steps:

  1. Call createTransactionForItem:quantity:comment:withCallbackQueue:onComplete: (MBBankDebit-p), which creates a transaction and presents a purchase screen to the user. If the user completes the purchase, the transaction's state changes from kMBTransactionStateNew to kMBTransactionStateAuthorized.
  2. Call openTransaction:withCallbackQueue:onComplete: (MBBankDebit-p), which changes the transaction's state to kMBTransactionStateOpen and places the user's funds into escrow.
  3. If the user's funds are successfully placed into escrow, deliver the virtual item to the user.
  4. Call closeTransaction:withCallbackQueue:onComplete: (MBBankDebit-p) to capture the user's funds and change the transaction's state to kMBTransactionStateClosed.

Important: Your app must wait for the callback from each method and verify that the request succeeded before proceeding to the next step.

Call this method if any of the following occur:

  • The user does not have sufficient funds for the transaction.

Continue processing a transaction created by an app server using the Mobage REST API.

When a transaction is created through the REST API, its state is set to kMBTransactionStateNew. Calling this method on the transaction does the following:

  1. The transaction's state is set to kMBTransactionStateAuthorized.
  2. The user's funds are placed into escrow.
  3. The transaction's state is set to kMBTransactionStateOpen. The virtual item should now be delivered to the user.

If the user has insufficient funds for the transaction, the transaction's state is set to kMBTransactionStateCanceled.

Note: Before you call this method, you must use the transaction ID to retrieve an MBTransaction object. Call getTransaction:withCallbackQueue:onComplete: to retrieve the object.

Create a new transaction.

When your app calls this method, the following steps occur:

  1. The app's inventory is checked to make sure the requested item exists, and a new transaction is created, with its state set to kMBTransactionStateNew.
  2. Mobage presents a screen that prompts the user to confirm the transaction.
  3. If the user decides not to proceed with the transaction, the transaction is automatically canceled. Otherwise, the transaction's state is set to kMBTransactionStateAuthorized.

quantityThe quantity of the item data.
