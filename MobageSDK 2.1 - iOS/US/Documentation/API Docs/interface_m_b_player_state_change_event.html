Detailed Description

Stores information about an event that describes changes in a user's state.

For example, your app can generate an MBPlayerStateChangeEvent when the user gains or loses a life, or when the user advances to a new level. Your app should also generate an MBPlayerStateChangeEvent when a user purchases or spends virtual currency.

Call report: (MBEventReporter-p) to report the event to the analytics server.

Create an event that describes changes in a user's state.

Parameters:
eventIdThe event ID.
changeInformation about the change that has occurred in the user's state. Include only properties that have changed in a way that you want to track. In addition, report the change in the property's value, not the new value of the property. For example, if the user spent 100 units of currency, use the value -100. Each value must be an NSString.
playerStateInformation about the user's current state, such as the user's level and number of lives remaining, or nil to omit this information. Each value must be an NSString. Note: Do not use this method to report events that primarily affect the user's state. Also, do not include information that is unique to the user, such as a user ID or nickname.
Returns:
The new analytics event.
